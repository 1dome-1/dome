# Coding Standard

## Style
- Prefer clear, readable JavaScript over clever abstractions.
- Keep functions focused on single responsibilities.
- Use consistent naming and indentation.

## Game Development Rules
- Keep state updates predictable and centralized.
- Avoid breaking browser compatibility for simple use cases.
- Preserve the single-file prototype structure unless a modular refactor is explicitly needed.

## Documentation
- Update documentation alongside major gameplay changes.
- Keep docs concise and directly connected to the current implementation.
